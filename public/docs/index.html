Introduction

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

https://safaqatek.com

Authenticating requests

This API is authenticated by sending an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

register

If data was valid and everything is okay, you'll get a 200 OK response.

Otherwise, the request will fail with status 422 with validation error if exist .

Example request:
curl --request POST \
    "https://safaqatek.com/api/v1/en/user/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"username\": \"zhskuvcvza\",
    \"email\": \"ohoeger@example.org\",
    \"password\": \"rusrtu\",
    \"phone\": \"et\"
}"

const url = new URL(
    "https://safaqatek.com/api/v1/en/user/register"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "username": "zhskuvcvza",
    "email": "ohoeger@example.org",
    "password": "rusrtu",
    "phone": "et"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (422, validation error):


{
    "message": "The given data was invalid.",
    "errors": {
        "username": [
            "The username field is required."
        ],
        "email": [
            "The email field is required."
        ]
    }
}

Example response (200, new user):


{
    "id": 20,
    "token": "20|zojdQE05gbfH8S08Pg6gvOqn7cZjzeJisRSR5Sxu"
}

logout

requires authentication

If request have param ' token_id ' valid and everything is okay, you'll get a 200 OK response.

Example request:
curl --request POST \
    "https://safaqatek.com/api/v1/en/user/logout" \
    --header "Authorization: Bearer {YOUR_AUTH_KEY}" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"token_id\": \"20\"
}"

const url = new URL(
    "https://safaqatek.com/api/v1/en/user/logout"
);

const headers = {
    "Authorization": "Bearer {YOUR_AUTH_KEY}",
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "token_id": "20"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200, logout):


{
    "message": "you are logout now"
