Introduction

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

http://safaqatek.test

Authenticating requests

To authenticate requests, include an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

User management

APIs for managing users

login

If data was valid and everything is okay, you'll get a 200 OK response.

Otherwise, the request will fail with status 422 with validation error if exist .

Example request:
curl --request POST \
    "http://safaqatek.test/api/v1/en/user/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"xhill@example.com\",
    \"password\": \"est\"
}"

const url = new URL(
    "http://safaqatek.test/api/v1/en/user/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "xhill@example.com",
    "password": "est"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (422, validation error):


{
    "message": "The given data was invalid.",
    "errors": {
        "message": "The given data was invalid.",
        "errors": {
            "data": [
                "The provided credentials are incorrect."
            ]
        }
    }
}

Example response (200, login):


{
    "id": 20,
    "token": "20|zojdQE05gbfH8S08Pg6gvOqn7cZjzeJisRSR5Sxu"
